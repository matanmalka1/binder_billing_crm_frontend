# אפיון מסכים וסידור מחדש

## מטרת הסידור

- להפוך את המערכת לכלי עבודה יומי ברור לצוות משרד.
- להפריד בין עבודה שוטפת, לקוחות, מסים, דוחות והגדרות.
- לייצר מבנה מסך אחיד: כותרת, פעולות, מדדים, סינון, טבלה, Drawer/Modal.
- לשמור על הרשאות: `ADVISOR` מבצע פעולות כתיבה, `SECRETARY` עובד בעיקר בצפייה ותפעול.

## תבנית מסך אחידה

כל מסך רשימה צריך להופיע בסדר הבא:

1. `PageHeader`
   - כותרת בעברית.
   - תיאור קצר רק כשיש ערך תפעולי.
   - פעולות ראשיות בצד שמאל: יצירה, ייבוא/ייצוא, פעולה רוחבית.
2. Alert הרשאות
   - מוצג רק כשמשתמש מוגבל בפעולות כתיבה.
3. מדדי על
   - עד 4 כרטיסים.
   - לחיצה על מדד מפעילה סינון.
4. `FilterPanel`
   - חיפוש, סטטוס, תאריכים, בעל טיפול, סוג חובה.
   - reset ברור.
5. טבלה
   - `PaginatedDataTable` למסכי רשימה.
   - לחיצה על שורה פותחת detail route או drawer לפי עומק העבודה.
6. Empty state
   - `StateCard` או empty state של הטבלה, עם פעולה אחת ראשית.
7. Drawers/Modals
   - יצירה ועריכה ב־`FormModal`.
   - פרטי משנה ב־`DetailDrawer`.
   - פעולות row בתוך dropdown ברשימה, inline בתוך drawer.

## תבנית מסך פרטים

1. `PageHeader` עם breadcrumbs.
2. תגיות סטטוס ומידע מזהה בשורת metadata.
3. Tabs קבועים לפי הישות.
4. פעולת עריכה ראשית אחת בכותרת.
5. פעולות מסוכנות או סגירה רק באזור ייעודי בתוך הטאב הרלוונטי.

## כללי הרשאות UI

| פעולה | ADVISOR | SECRETARY |
| --- | --- | --- |
| צפייה | כן | כן |
| יצירת לקוח | כן | לא |
| עריכת לקוח | כן | לא |
| מחיקה/שחזור | כן | לא |
| שינוי סטטוס עבודה | כן | מוגבל לפי service |
| ייבוא/ייצוא | כן | לפי endpoint |
| ניהול משתמשים | כן | לא |

בכל מסך שבו פעולה מוסתרת ל־`SECRETARY`, יש להציג Alert קצר בעברית אם ההגבלה משפיעה על העבודה במסך.

## החלטות מוצר פתוחות

- `tasks`: אם אין מסך route, לא להציג בניווט. אם יש צורך, להגדיר כמסך "משימות" תחת עבודה שוטפת.
- `taxProfile`: לשלב בתוך עסק/לקוח, לא כמסך עצמאי, אלא אם נדרש workflow ייעודי.
- `notifications`: להשאיר כ־bell/dropdown. מסך רשימה רק אם יש טיפול בהיסטוריה או resend.
- `taxDashboard`: לשלב בלוח הבקרה הראשי, לא להוסיף dashboard מסים נפרד בשלב זה.

## סדר ביצוע מומלץ

1. לעדכן ניווט ונתיבים לדוחות ניהול.
2. ליישר את כל מסכי הרשימה לתבנית האחידה.
3. להחליף filter bars ב־`FilterPanel`.
4. לאחד `SelectDropdown` לתוך `Select`.
5. לאחד modals תחת `FormModal`.
6. לאחד drawers תחת `DetailDrawer`.
7. להוסיף דוחות חסרים רק אחרי שהניווט והמסכים הקיימים יציבים.
